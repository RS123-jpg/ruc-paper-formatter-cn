# ruc-paper-formatter-cn

面向中文课程论文、比赛论文和一般投稿稿件的 DOCX 排版 Skill。

本项目以《中国人民大学学报》体例为基础，并针对课程论文、比赛论文、Word/WPS 兼容性及多 AI 平台使用场景进行了适配。

> 本项目为个人开发的非官方工具，与中国人民大学及《中国人民大学学报》编辑部不存在隶属、授权或合作关系。

## 主要功能

- 统一页面、页边距、字体、字号、行距和段落格式
- 规范主标题、副标题、作者信息和多级标题
- 支持 `一、`、`（一）`、`1.`、`（1）` 四级标题
- 统一摘要与关键词格式
- 缺失摘要或关键词时，可依据全文补全并明确标注
- 将中文正文中的英文直引号转换为中文引号
- 处理正文上标、Word 脚注和文献引用
- 支持“页底脚注 + 文末参考文献”并存
- 统一图表题注、编号和交叉引用
- 文末参考文献独立分页
- 修复部分 Word/WPS 标题错位和重复编号问题
- 输出排版后的 DOCX 和简明修改报告

## 默认格式

标题层级：

- 一级标题：`一、一级标题`
- 二级标题：`（一）二级标题`
- 三级标题：`1. 三级标题`
- 四级标题：`（1）四级标题`

摘要与关键词：

- `［摘要］摘要正文`
- `［关键词］关键词1；关键词2；关键词3`

人大校名标志页眉为可选功能，默认不添加。

## 支持平台

本项目提供面向以下平台的说明或适配文件：

- ChatGPT
- Codex
- Gemini
- WorkBuddy
- Claude
- 其他支持自定义指令、文件读取或脚本执行的 AI Agent

不同平台的文件访问和 Python 执行能力不同，因此不保证所有平台都可以直接修改 DOCX。

## 下载

请在 GitHub Releases 页面下载最新版 `skill.zip`。

该压缩包用于分享和导入，不会自动安装，也不会主动修改本地已有的 Skill。

## 使用说明

能够读取项目文件并执行脚本的 AI，可以直接读取根目录中的 `SKILL.md`。

不能直接执行脚本的平台，可以使用 `platforms` 目录中的对应说明，或采用通用提示词和便携工作流。

## 已知限制

- Word 与 WPS 的分页和字体替换结果可能存在差异
- 复杂公式、文本框和嵌套表格可能需要人工复核
- 无法确定来源的手工角标不会被强行匹配
- Word 中的目录、页码和交叉引用有时需要手动更新域
- 本工具不能替代学校、比赛主办方或期刊发布的正式模板

## 项目结构

- `SKILL.md`：Skill 核心说明
- `scripts/`：DOCX 处理脚本
- `references/`：排版规范和规则文件
- `assets/`：可选页眉等资源
- `platforms/`：不同 AI 平台适配说明
- `dist/skill.zip`：对外分享安装包
